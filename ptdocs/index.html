RESEARCH

My research aims at ensuring cyber environments free of malicious activities through principled schemes. A typical cyber environment consists of two parts, the runtime platform (e.g., the Windows Operating System, or the browser platform) and the executables (e.g., software programs, mobile Apps, or JavaScript libraries) running on top of the platform. My research drives security innovations in both parts of cyber environments, specifically, designing new security primitives for runtime platforms, and developing new analysis methods to prevent attacks exploiting vulnerabilities in executables.

For new security primitives in runtime platforms, I have been investigating security primitives that guarantee the computation of executables is free of unexpected results. Specifically, my research in this direction focuses on the following aspects:

    • In a cyber environment, what is the uniqueness in its runtime platform and what are the appropriate primitives to eliminate unexpected results? 
    • By reflecting on the experience in designing security primitives in one platform, what can we learn from them to improve security of other platforms? 

For new methods of analyzing executables, I aim to develop methods for accurate analysis, which can significantly improve the capability for attack response and prevention. Specifically, I advance my research in this area from the following perspectives:

    • (Diagnosis) When an attack is detected, how to efficiently and accurately identify the key characteristics of the attack?
    • (Mitigation) With the attack characteristics, how to promptly prevent ongoing attacks to minimize their impact?
    • (Prevention) How to identify and fix software vulnerabilities through accurate analysis of programs?

TEACHING

For the past five years in NUS, I actively participated in building and restructuring the cyber security curriculum. In particular, I led the effort of creating and redesigning courses in system security. Before I joined the department, existing security courses focused on cryptography and network security. From 2008, I created or redeveloped a series of courses. Specifically, I have created two new courses, and redevelop the syllabus of another course.

I created a new undergraduate course, CS4238 Computer Security Practice (Sample Syllabus), to train the system security skills of senior undergraduate students, preparing them for a system security career. To support the instruction of this course, I designed the layout and networking of a new teaching lab, supporting 40 students (10 project groups). I also created a new course, CS5231 Systems Security. It focuses on system security knowledge and skills, as well as latest developments in system security research.

I also redeveloped the undergraduate course CS3235 (Introduction to Computer Security), shifting its focus from coding theory and cryptography to general computer security (including basic cryptography, software security, system security, network security, and web security).

I have been practicing a new method of teaching security principles through practice. I designed a series of live demos in class, showing students complex concepts like buffer overflow attacks and rootkit analysis. I also distribute virtual machines and external hard drives to students for them to study security tools. The method is effective in motivating students to see the connections between security knowledge and real-world problems.
